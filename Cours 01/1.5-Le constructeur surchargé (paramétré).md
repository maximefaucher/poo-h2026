# Le constructeur surchargé (paramétré)

## 1. Définition du constructeur surchargé

En Java, une classe peut posséder plusieurs constructeurs. On parle alors de **surcharge de constructeurs**.

L'idée est simple :

> Plusieurs constructeurs portant le **même nom** (celui de la classe), mais avec des **paramètres différents**.

Les paramètres seront différents en **type**, en **nombre** ou en **ordre**, conformément avec la définition de la **surcharge**.

Pourquoi faire ça?

- Pour offrir **plusieurs façons de créer un objet**.
- Pour **initialiser l'objet avec des valeurs différentes** selon les besoins.
- Pour **simplifier le code du programmeur** lors de l'utilisation de la classe.

Il est ainsi possible d'instancier...

- Un objet créé avec des **valeurs par défaut**
- Un objet créé avec des **valeurs fournies par l'utilisateur** (multiple)

## 2. L'objet `this`

Le mot-clé `this` représente **l'objet courant**, c'est-à-dire l'objet qui est en train d'être construit ou qui exécute une méthode.

Dans le contexte des constructeurs, `this` sert principalement à :

1. **Distinguer un attribut d'un paramètre** qui porte le même nom.

    ```java
    public Etudiant(String nom) {
        this.nom = nom; // this.nom = l'attribut / nom = le paramètre
    }
    ```

2. Appeler un **autre constructeur de la même classe**.

    ```java
    public Rectangle() {
        this(1, 1); // Appelle un constructeur paramétré
    }
    ```

    On appelle cela **l'enchaînement (le chaînage) de constructeurs**.

## 3. L'implémentation du constructeur surchargé

Voici un exemple complet d'une classe avec plusieurs constructeurs :

```java

public class Voiture {

    public String marque;
    public String modele;
    public int annee;

    // Constructeur par défaut
    public Voiture() {
        this("Honda", "Civic", 2020); 
    }

    // Constructeur avec deux paramètres
    public Voiture(String marque, String modele) {
        this(marque, modele, 2020); 
    }

    // Constructeur avec trois paramètres
    public Voiture(String marque, String modele, int annee) {
        this.marque = marque;
        this.modele = modele;
        this.annee = annee;
    }
}
```

Points importants à retenir :

- Tous les constructeurs portent le **même nom que la classe**.
- Ils doivent avoir des **signatures différentes** (signature = nom + liste de paramètres).
- Un appel à `this(...)` doit être la **première instruction du constructeur**, si cet appel est utilisé. Cela permet d'éviter la duplication de code.

## 4. L'appel au constructeur surchargé

Pour utiliser ces constructeurs, on choisit celui qui correspond aux paramètres fournis :

```java
Voiture v1 = new Voiture();                        // Appelle le constructeur par défaut
Voiture v2 = new Voiture("Toyota", "Corolla");     // Appelle le constructeur à 2 paramètres
Voiture v3 = new Voiture("Ford", "Mustang", 1967); // Appelle le constructeur à 3 paramètres
```

Chaque instruction crée un **objet différent selon les valeurs initiales désirées**.

---

## EN RÉSUMÉ

- Un **constructeur surchargé** = plusieurs façons de créer un objet avec des valeurs différentes.
- `this` = **l'objet courant** (utile pour clarifier ou appeler un autre constructeur).
- L'enchaînement des constructeurs avec `this(...)` évite la duplication de code.
- L'utilisateur choisit le constructeur en fonction des paramètres fournis.
