# La programmation orientée objet (POO)

La **programmation orientée objet** (POO) est un paradigme en programmation informatique qui organise le code autour de structures appelées objets.

Références :

[Page Wikipedia sur la POO](https://fr.wikipedia.org/wiki/Programmation_orient%C3%A9e_objet).

## 1. Le modèle objet : définitions

### OBJET

Un **objet** est une entité qui regroupe :

- Des propriétés (appelées **attributs**) : caractéristiques de l'objet.
- Des comportements (appelés **méthodes**) : actions que l'objet peut effectuer ou subir.

### CLASSE

Une **classe** est un modèle ou une « recette » qui définit la structure des objets (leurs attributs et méthodes).

#### Exemples d'objets basés sur la vie courante

- OBJET 1 = Voiture

  - CARACTÉRISTIQUES (attributs) = marque, modèle, année, couleur, en marche, position, propriétaire
  - ACTIONS (méthodes) = démarrer, arrêter, déplacer, acheter, vendre

- OBJET 2 = Porte

  - CARACTÉRISTIQUES (attributs) = est fermée, est barrée
  - ACTIONS (méthodes) = ouvrir, fermer, barrer, débarrer

## 2. La modélisation (UML)

Le **UML** (*Unified Modeling Language*, ou langage de modélisation unifié) est un **langage graphique standardisé** utilisé en informatique pour concevoir, visualiser et documenter des systèmes logiciels.

Il ne s'agit pas d'un langage de programmation, mais d'un **ensemble de diagrammes qui servent à représenter la structure et le comportement d'un système** avant (ou pendant) son développement.

Référence : [page Wikipedia sur le UML](https://fr.wikipedia.org/wiki/UML_(informatique)).

### Utilités du UML

- Mieux **comprendre un système** complexe
- **Communiquer clairement** entre analystes, développeurs et clients
- **Planifier une application** avant de la coder
- **Documenter** un logiciel existant

### Principaux types de diagrammes UML

- Diagramme de **cas d'utilisation** (*use cases*)  
  **Ce que fait le système** du point de vue utilisateur.  
  *Exemples* : se connecter, ajouter un employé, etc.
- Diagramme de **classes**  
  Montre la **structure du programme** : classes, attributs, méthodes, relations (héritage, association, composition, etc.).
- Diagramme de **séquence**  
  Montre **l'ordre des interactions** entre objets dans le temps : qui appelle quoi, et quand?
- Diagramme d'**activités**  
  Ressemble à un **organigramme** : décrit le flux d'exécution d'un processus.
- Diagramme d'**états**  
  Montre les **états possibles d'un objet** et les transitions
  *Exemple* : Commande : créée → payée → expédiée

### Exemples de diagrammes de classes UML

Voici quelques exemples en notation simple : on représente une classe par un **rectangle divisé en 3 parties** :

1. Le **nom de la classe** en haut
2. La liste des **attributs** (caractéristiques de l'objet, avec des noms usuels) au centre
3. La liste des **méthodes** (actions possibles avec ou sur l'objet, avec des noms usuels) au bas

## 3. La syntaxe Java d'une classe

La syntaxe générale de la déclaration d'une classe Java est la suivante :

1. La **déclaration de la classe suivie d'accolades** `{ }` marquant le début et la fin du code de la classe
2. Dans le haut du code de la classe, on retrouve la **liste des attributs** des objets de la classe
3. Ensuite on retrouve la **liste des méthodes** de la classe

```java
//////////////////////////////
// 1. Déclaration de la classe
public class MaClasse {

    /////////////////////////
    // 2. Liste des attributs
    public int entier;
    public String chaine;
    public boolean booleen;
    // etc.

    ////////////////////////
    // 3. Liste des méthodes
    public void methode1() {
        // code de la méthode.
        // les attributs de l'objet sont accessibles ici (selon les accès)
    }
    public void methode2() { 
        // code de la méthode.
        // les attributs de l'objet sont accessibles ici (selon les accès)
    }
    public void methode3() { 
        // code de la méthode.
        // les attributs de l'objet sont accessibles ici (selon les accès)
    }
    // etc.

}
```

Voyons cette syntaxe appliquée à nos 2 classes...

### La classe `Voiture`

```java
public class Voiture {

    // Attributs
    public String marque;
    public String modele;
    public int annee;
    public String couleur;
    public boolean enMarche;
    public double position; 
    public String proprietaire;

    // Méthodes
    public void demarrer() {
        enMarche = true;
    }

    public void arreter() {
        enMarche = false;
    }

    public void deplacer(double _position) {
        if(enMarche)
            position = _position;
    }

    public void vendre(String _acheteur) {
        proprietaire = _acheteur;
    }

}
```

### La classe `Porte`

```java
public class Porte {

    // Attributs
    public boolean estOuverte;
    public boolean estBarree;

    // Méthodes
    public void ouvrir() {
        estOuverte = true;
    }

    public void fermer() {
        estOuverte = false;
    }

    public void barrer() {
        estBarree = true;
    }

    public void debarrer() {
        estBarree = false;
    }

}
```
